Here are a few brainstormed ideas for single-file Python projects that fit your constraints perfectly. Since you are limited to `print()`, `input()`, variables, and basic math, the challenge lies in **logic flow** (using `if/elif/else` and loops) and **data manipulation** rather than complex libraries.

I have categorized them by theme so you can choose what interests you most.

### 1. The "Retro" Text Adventure (Interactive Story)
Instead of calculating numbers, you guide the user through a story where their choices change the outcome.
*   **Concept:** The user is trapped in a room, a forest, or a spaceship. They must make binary choices (Go Left/Right, Fight/Run).
*   **Key Mechanics:**
    *   Use a **variable** to store the user's "Health" or "Gold" (integers).
    *   Use `if/elif` to check conditions (e.g., `if health <= 0: print("You died.")`).
    *   Use a `while True` loop to keep the story going until the user quits or wins.
*   **Why it fits:** It relies heavily on text strings and simple integer math (subtracting health), but it feels like a game.

### 2. The "Unit Converter" with Validation
A classic tool, but with a twist: it must handle bad input gracefully.
*   **Concept:** Convert between different units (e.g., Celsius to Fahrenheit, Miles to Kilometers, Hours to Minutes).
*   **Key Mechanics:**
    *   Ask the user for the value (float) and the source unit (string).
    *   Perform the math (e.g., `(C * 9/5) + 32`).
    *   **The Twist:** Use a `try...except` block (or simple logic) to ensure the user actually types a number. If they type "apple", print an error message and ask again.
*   **Why it fits:** It reinforces the difference between integers and floats and teaches the importance of user input validation.

### 3. The "Tip Calculator" (Social Scenario)
Similar to your budget calculator but focused on splitting bills among friends.
*   **Concept:** The user enters the total bill amount, the number of people, and the desired tip percentage.
*   **Key Mechanics:**
    *   Calculate the subtotal per person.
    *   Calculate the tip amount per person.
    *   Calculate the final total per person.
    *   Format the output to show 2 decimal places (using string formatting like `f"${:.2f}"`).
*   **Why it fits:** It uses floats extensively and demonstrates how to break down a complex real-world problem into small math steps.

### 4. The "Number Guessing Game"
A game where the computer picks a number (or the user picks one, and the computer guesses), but since we can't use random libraries, the computer must guess using logic.
*   **Concept:** The user thinks of a number between 1 and 100. The program asks "Is your number higher or lower than 50?" and adjusts its guess accordingly.
*   **Key Mechanics:**
    *   Start with a middle number.
    *   Ask the user for input ("higher", "lower", or "correct").
    *   Update the search range using math (add or subtract based on input).
    *   Repeat until the user says "correct".
*   **Why it fits:** It teaches logic flow and conditional branching without needing any external libraries.

### 5. The "BMI (Body Mass Index) Checker"
A health utility that gives feedback based on a calculated score.
*   **Concept:** Ask for weight (kg) and height (m). Calculate BMI = weight / (height * height).
*   **Key Mechanics:**
    *   Check the resulting float against ranges.
    *   `if bmi < 18.5: print("Underweight")`
    *   `elif bmi < 25: print("Normal")`
    *   `else: print("Overweight")`
    *   Add a loop so the user can calculate for multiple people without restarting the script.
*   **Why it fits:** It is a perfect example of using `if/elif/else` chains with float variables.

### 6. The "Simple Quiz App"
A study tool where the user answers multiple-choice questions.
*   **Concept:** Ask 5 questions about a specific topic (e.g., Python syntax, History, or Science).
*   **Key Mechanics:**
    *   Store the correct answer in a variable.
    *   Ask the user to input their choice.
    *   Compare the user's input string to the correct string.
    *   Keep a score variable (integer) and update it (`score += 1`) for every correct answer.
    *   Print the final score at the end.
*   **Why it fits:** It combines string comparison with integer math (scoring) and provides immediate feedback.

### Recommended Structure for Your Code
Regardless of which idea you pick, structure your single file like this to look professional:

```python
# 1. Define variables for the game state or data
score = 0
health = 100
user_name = ""

# 2. Get initial input
user_name = input("Enter your name: ")
print(f"Welcome, {user_name}!")

# 3. Main Logic Loop (Optional but recommended)
while health > 0:
    # Ask for input
    choice = input("What do you do? (fight/escape): ")
    
    # Perform logic and math
    if choice == "fight":
        damage = 10 # Example integer
        health -= damage
        print(f"You fought! Health is now {health}.")
    elif choice == "escape":
        print("You ran away safely.")
        break # Stop the loop
        
    # Check end condition
    if health <= 0:
        print("Game Over!")
        break

# 4. Final Output
print(f"Thanks for playing, {user_name}.")
```
